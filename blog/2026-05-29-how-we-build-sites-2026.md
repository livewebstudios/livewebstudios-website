## The Stack We Used to Build With

For most of Live Web Studios' history, the answer to "how do we build this?" was WordPress. Pick a theme, customize it, install Divi or another page builder, add plugins for forms, SEO, caching, security, backups. Done.

It works. We've built hundreds of sites that way. But over time, the weight adds up.

## The Stack We Build With Now

Here's what a new LWS site looks like under the hood in 2026:

**Design phase:** We use Google Stitch and Claude Design to translate reference sites and design direction into real layout decisions — fast, iterative, without starting from a blank canvas every time.

**Build phase:** Claude Code and Cursor handle the actual site build. Clean HTML, CSS, and JavaScript written to your specs, not assembled from a template library.

**Content management:** For clients who need to edit their own content, we set up Decap CMS — a lightweight content editor that lives in the browser. You log in, edit your text, hit save. No database, no WordPress admin, no plugin updates.

**Forms:** Formspree handles contact forms cleanly and reliably. No plugin, no backend code.

**Hosting:** Sites deploy through GitHub to Netlify. Every change is versioned. Rollback is one click. Hosting is distributed on a global CDN, which means your site loads fast for visitors whether they're in Newark or Nevada.

## What You Don't Have Anymore

You don't have a database that can get corrupted. You don't have 40 plugins that need updating. You don't have a single server that goes down and takes your site with it.

## What You Do Have

A fast, clean, version-controlled site that costs less to host, loads faster for visitors, and requires far less maintenance to keep running right.
